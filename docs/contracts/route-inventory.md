# Legacy Route Inventory

This document inventories the route families expected from the legacy LIAN backend.
Each family groups semantically related endpoints. Use this as the source of truth
for route parity tracking.

> **Do not edit this file to add routes from memory.** Verify against the legacy
> backend's actual route definitions before adding entries.

## Route Family Definitions

### AUTH -- Authentication

| Method | Path                  | Purpose                       | Notes |
|--------|-----------------------|-------------------------------|-------|
| POST   | /api/auth/login       | Email/password login          | Returns session token |
| POST   | /api/auth/register    | User registration             | |
| POST   | /api/auth/logout      | Invalidate session            | |
| GET    | /api/auth/me          | Current user profile          | Requires auth |
| POST   | /api/auth/password    | Password change               | Requires auth |

### USERS -- User Management

| Method | Path                       | Purpose                       | Notes |
|--------|----------------------------|-------------------------------|-------|
| GET    | /api/users/:uid            | Get user by ID                | |
| GET    | /api/users/:uid/profile    | Full profile with fields      | |
| PUT    | /api/users/:uid            | Update user                   | Owner or admin |
| GET    | /api/users/:uid/posts      | User's posts                  | Paginated |
| GET    | /api/users/:uid/topics     | User's topics                 | Paginated |

## How to Use This Inventory

1. Each family maps to one or more Nest modules in the new backend.
2. Track migration status per family in `route-parity-tracker.md`.
3. Acceptance criteria per family live in `acceptance-criteria.md`.
4. The `check-route-parity.js` script can parse this file to report coverage.

## Adding New Routes

When discovering routes not listed here:

1. Verify the route exists in the legacy backend (not a guess).
2. Assign it to the correct family (or create a new family row).
3. Add a row to this table with method, path, purpose, and notes.
4. Update `route-parity-tracker.md` to mark it as unmigrated.
