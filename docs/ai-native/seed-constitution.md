# Seed Constitution

Immutable rules for the AI development control plane. This is the docs
mirror of the authoritative policy at
[.github/ai-policy/seed-constitution.md](../../.github/ai-policy/seed-constitution.md).

No automation, worker, orchestrator, or script may self-expand, override,
or relax these boundaries. Changes require a human-authored PR reviewed
by the architecture-review role.

---

## 1. High-Risk Human-Required Boundaries

The following operations MUST NOT be performed by any AI worker or
automation without explicit human approval in the issue or PR:

| Boundary | Why |
|----------|-----|
| Deleting or rotating secrets, tokens, or credentials | Irreversible exposure risk |
| Modifying `.env`, `.env.*`, or CI/CD pipeline definitions | Production blast radius |
| Changing Prisma schema or running migrations | Data integrity |
| Force-pushing to `main` or protected branches | History loss |
| Modifying `package.json` dependencies (add/remove) | Supply chain risk |
| Altering the seed constitution file | Self-expansion guard |
| Modifying `.github/ai-policy/**` | Policy integrity |
| Changing the launch gate, health gate, or batch launcher scripts | Control-plane integrity |

Workers encountering a task that touches any of these boundaries MUST
comment on the issue requesting human intervention.

---

## 2. Explicit Merge Allowlists

Workers may only merge PRs that touch files within their declared
`allowedFiles` boundary.

1. **No implicit broadening.** Fixing adjacent issues outside
   `allowedFiles` is not permitted, even if trivial.
2. **No self-granted extensions.** Workers MUST NOT edit task JSON or
   control-plane metadata to widen their own boundaries.
3. **Shared locks are explicit.** Access to shared resources requires a
   declared `sharedLocks` entry in the task contract.
4. **Docs-only tasks stay in docs.** Tasks with `allowedFiles`
   exclusively under `docs/` MUST NOT edit source, scripts, or config.

---

## 3. Main-Red Launch Stop

When the main branch health state is **red**:

1. No new feature or runtime workers may launch.
2. In-flight workers MUST be paused.
3. Recovery workers take priority.
4. The red-state block is absolute — no override exists.

See [main-health-policy.md](main-health-policy.md) for the full health
state matrix and in-flight handling rules.

---

## 4. Legacy Backend Read-Only Policy

The legacy backend is **read-only** for AI workers:

1. Workers MAY read legacy code for context.
2. Workers MUST NOT modify, refactor, or delete legacy files.
3. Migration work requires a dedicated task with human approval.
4. Only the migration matrix updater may propose legacy changes, and
   those require human review.

---

## 5. No Worker Scope Expansion

A worker's scope is immutable after launch:

1. **No self-expansion.** Workers MUST NOT modify their task JSON,
   `allowedFiles`, `conflictGroup`, or `sharedLocks`.
2. **No transitive expansion.** Dependencies outside `allowedFiles` are
   blockers, not scope extensions.
3. **No orchestration self-promotion.** Workers MUST NOT spawn
   sub-workers or modify scheduling state.
4. **No policy modification.** Workers MUST NOT edit files under
   `.github/ai-policy/`, `.github/ai-state/`, or `docs/ai-native/`
   unless explicitly in `allowedFiles`.

---

## Enforcement

| Rule | Enforced By | When |
|------|-------------|------|
| High-risk boundaries | Boundary guard + worker honor system | Pre-merge validation |
| Merge allowlists | Boundary guard (`allowedFiles` check) | Pre-merge validation |
| Main-red launch stop | Launch gate (`check-launch-gate.ps1`) | Pre-launch |
| Legacy read-only | Boundary guard + migration matrix | Pre-merge validation |
| No scope expansion | Worker contract + task JSON immutability | Runtime |

---

## Amendment Process

These rules may only be changed by:

1. A human-authored PR (not generated by automation).
2. Review by the `architecture-review` role.
3. Approval by the repository owner.
4. A corresponding update to the authoritative file at
   [.github/ai-policy/seed-constitution.md](../../.github/ai-policy/seed-constitution.md).
